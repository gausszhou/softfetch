## 3. 性能优化

### 3.1 并行检测

**目标：** 将串行检测改为并行检测，提升检测速度

**实现方案：**

```go
func Detect(detectors ...Detector) DetectionResult {
    result := DetectionResult{
        Tools: make([]Tool, 0, len(detectors)),
    }
    
    // 并行执行所有检测器
    resultChan := make(chan Tool, len(detectors))
    var wg sync.WaitGroup
    
    for _, detector := range detectors {
        wg.Add(1)
        go func(d Detector) {
            defer wg.Done()
            resultChan <- d.Detect()
        }(detector)
    }
    
    // 等待所有检测完成
    go func() {
        wg.Wait()
        close(resultChan)
    }()
    
    // 收集结果
    for tool := range resultChan {
        result.Tools = append(result.Tools, tool)
    }
    
    return result
}
```

**性能提升预期：**
- 串行检测：O(n * timeout)，每个工具依次检测
- 并行检测：O(timeout)，所有工具同时检测
- 假设每个检测耗时 100ms，12 个工具：
  - 串行：约 1200ms
  - 并行：约 100ms
- **预计提升 10 倍以上**

### 3.2 优化说明

- 使用 `sync.WaitGroup` 管理并发
- 使用 channel 收集结果，保证结果顺序
- 超时控制已在 command.Execute 中实现
- 并行检测不影响原有错误处理逻辑
